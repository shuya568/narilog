# narilog – Project Overview

本ドキュメントは、本プロジェクト（個人ブログ兼ポートフォリオ開発）の  
**目的・背景・進め方・成果物** を明確化し、AI と人間の間でブレのない共通認識を作るためのものです。

---

## 🎯 1. プロジェクトの目的

本プロジェクトの目的は大きく 2 つある。

### **1. フルスタック TypeScript によるポートフォリオの構築**

NestJS（Backend）・Next.js（Frontend）・Prisma（ORM）・GCP（Cloud Run / Cloud SQL）を用いて  
**実際のサービスとして成立するブログアプリケーション**を構築することで、  
フルスタックエンジニアとしてのスキルセットを可視化する。

### **2. 自作サービスを運用する経験を積む**

これまで実務では大規模システムの開発・運用経験があるが、  
「自分で企画 → 設計 → 実装 → デプロイ → 運用」  
の一連の流れを経験する機会が少なかった。

本プロジェクトでは、  
**自ら作ったサービスを継続運用する** ことを重要な目的とし、  
以下を体験する：

- 機能追加・保守・改善サイクル
- インフラ運用（Cloud Run / Cloud SQL / Cloud Storage）
- デプロイ/バージョニング戦略
- 障害対応プロセス（ログ、アラート、ロールバック）

---

## 🛠 2. プロジェクトの進め方

本プロジェクトでは、ChatGPT・Gemini などの生成 AI を積極的に活用し、  
**AI と人間が対話しながら進める “共同開発スタイル”** を採用する。

フェーズごとに 1 つのドキュメントを作成し、  
**「フェーズ単位の共通認識」を AI と人間が共有する** ことを最優先とする。

---

### 📌 フェーズ構成（各フェーズで 1 つの公式ドキュメントを作成）

1. **要件定義（Requirements Definition）**

   - 機能要件 / 非機能要件
   - UI 要件・運用要件
   - 認証・権限・投稿フロー
   - 将来拡張の整理

2. **データモデル定義（Domain Modeling with DDD）**

   - ドメイン境界づけ
   - エンティティ / 値オブジェクト / 集約ルート
   - 状態遷移
   - Prisma スキーマへの投影

3. **アーキテクチャ設計（Architecture Design）**

   - NestJS/Next.js の責務分離
   - API 設計（REST or GraphQL）
   - 認証（まずは NextAuth → 後に JWT）
   - デプロイ構成（Cloud Run / Cloud SQL）

4. **実装フェーズ（Implementation）**

   - リポジトリ構成
   - エディタ（Milkdown）
   - Prisma + NestJS のドメイン実装
   - Next.js の管理画面

5. **テスト＆運用フェーズ（Test & Operations）**
   - E2E テスト（NestJS）
   - 運用監視
   - バージョニング
   - 機能追加サイクルの確立

---

## 🤝 3. AI 活用方針

- 各フェーズで AI に質問し、  
  **「構造化されたドキュメント」** を作成してもらう
- コードも **可能な限り AI と共同で作成** し、改善サイクルを回す
- 曖昧な点は必ず質問し、決定事項はドキュメントに追記する
- DDD の理解不足部分は AI の補助で補う
